Chapter 8- Functions

Functions in C are essential for modularizing code, making it more organized and reusable. They allow you to break down a program into smaller, manageable parts. Here's an overview

Function Declaration:

A function in C typically consists of a function prototype (declaration) and its definition. The prototype specifies the function's name, return type, and parameter types (if any).

// Function prototype return_type function_name(parameter_type1, parameter_type2, ...);

Function Definition:

The function definition contains the actual code that the function executes. It includes the function header and the body enclosed in curly braces {}.

// Function definition return_type function_name(parameter_type1 param1, parameter_type2 param2) { // Function body // Perform operations using param1 and param2 return result; // Return value of type return_type }

// Function prototype int add(int a, int b); // Function definition int add(int a, int b) { int sum = a + b; return sum; }

Function Components

Return Type: Specifies the type of data that the function will return. If a function doesn't return anything, its return type is void.

Function Name: A unique identifier for the function.

Parameters: Values that the function can accept. Parameters are optional; a function can have none or multiple parameters.

Function Body: The actual code that the function executes, enclosed within curly braces {}.

Return Statement: Specifies the value to be returned by the function. The return statement exits the function and sends the value back to the calling code.

Function Calls: To use a function, you call it by its name and pass arguments (if required).

Function Prototypes:

Declaring the function prototype before its use allows the compiler to understand the function's signature before encountering its actual definition.

Function Scope:

Variables declared within a function have local scope, meaning they can only be accessed within that function. Parameters and variables declared in the function are typically destroyed once the function finishes executing.

Functions are a cornerstone of structured programming, enabling code reuse and making programs more organized and easier to maintain.
